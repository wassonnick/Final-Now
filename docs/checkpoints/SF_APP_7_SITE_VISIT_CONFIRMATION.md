# SF-APP-7 — Mobile Site Visit Confirmation

Date: 2026-07-22

Base commit before work:
- `6ba1a771 Build mobile account enquiries dashboard`

## What changed

- Added mobile support for confirming proposed site visit slots.
- Extended account dashboard site visit payload handling with:
  - `proposed_slots`
  - `confirmation_token`
- Added `confirmSiteVisit()` mobile API helper using:
  - `POST /api/site-visits/{token}/confirm`
- Updated `/my-enquiries` visit cards to:
  - show proposed visit slots as selectable chips
  - confirm the selected slot
  - refresh the account dashboard after confirmation
  - keep confirmed visits read-only

## Product and safety rules preserved

- Confirmation requires the backend-generated visit token.
- Users can only confirm one of the proposed slots accepted by the backend.
- Admin/ops workflow remains the source of proposed visit slots.
- No private lead/contact details are exposed in mobile.

## Validation

- `cd mobile && npm run typecheck` — passed
- `cd mobile && npm run lint` — passed

## Known remaining risks

- Mobile does not yet support proposing alternate slots.
- There is no push notification/reminder flow in mobile yet.
- Confirmation depends on active visit tokens generated by backend/admin ops.
